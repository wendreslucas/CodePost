Olá!

Bem-vindo ao teste de codificação do CodeLeap!

O objectivo é criar uma aplicação muito simples que irá executar funções básicas de CRUD.

Frontend
Se estiver a candidatar-se ao frontend, por favor sinta-se à vontade para escolher React ou React-Native para construir esta candidatura.

Backend
Se está a candidatar-se a backend, por favor use Django e DRF para construir esta aplicação. Não é necessário construir nenhum frontend/UI, apenas os pontos finais API para que a aplicação funcione como a documentação.

Por favor, sinta-se à vontade para utilizar quaisquer pacotes ou ferramentas que deseje, em complemento dos anteriores.

Conclusão e entrega
Por favor, carregue o seu código para um repositório ao qual possamos aceder e informe-nos quando estiver pronto. Se possível, por favor, implante o código em algum servidor remoto para que também esteja a funcionar.

Não se preocupe em ter tudo perfeito, o objectivo é certificar-se de que consegue lidar com as ferramentas e construir interfaces personalizadas, navegação, gestão do estado e ter os conhecimentos gerais de engenharia para resolver os problemas em mãos.


LOGIN
O primeiro ecrã é um tipo de ecrã do tipo "modal de inscrição".
Não é necessário criar utilizadores no backend. Basta manter a cadeia de nomes de utilizador algures no frontend para que possamos submetê-la ao criar um post mais tarde.

MAIN
Esta página contém várias características diferentes:
- Capacidade de criar mensagens
- Capacidade de ir buscar e ler mensagens
- Abrir alerta de eliminação (apenas os seus próprios artigos)
- Abrir editar modal (apenas os seus próprios artigos)

A lista de artigos deve ser actualizada automaticamente quando é criado um novo post.

Ordenar a lista pelo mais recente no top.post mais tarde.



DELETE
Ao apagar um item, mostrar um alerta de sistema para confirmar (não se preocupe de forma alguma com o estilo aqui).


EDIT
Auto-explicativo, faça um modal para editar os itens existentes na lista.




BUTTONS

Botão "Grey out" e desactivar se o campo do nome de utilizador estiver vazio

Botão "Grey out" e desactivar se algum campo estiver vazio

Estes só devem ser visíveis para si próprio (verificação simples de cordel)



CRITÉRIOS
Critérios de avaliação de fachada

Será avaliado de acordo com vários pontos diferentes:

Em geral, deverá desenvolver isto como se fosse dirigido a um consumidor final, e não a outros criadores. Tente impressionar o utilizador dentro das limitações do design, e não apenas os programadores.

O seu projecto deve ser mais ou menos parecido com estes desenhos aqui, e não será avaliado por fazer os desenhos radicalmente personalizados. No entanto, a coerência é fundamental, por isso certifique-se de que tudo tem um aspecto agradável e limpo.

A usabilidade é primordial. Certifique-se de que uma pessoa normal seria capaz de compreender e utilizar a interface facilmente em diferentes cenários, pontos de partida, dispositivos ou tamanhos de ecrã.

Atenção aos detalhes. Por favor, assegure-se de que olha cuidadosamente para todos os detalhes nesta página.

Mantenha-o simples. No seu código, será avaliado pela eficácia com que pode realizar esta tarefa. Estamos à procura de pessoas com objectivos e orientadas para resultados.

Se tiver algum problema ou questão, ou se algo não estiver a funcionar bem, por favor contacte victor@codeleap.co.uk


ORIENTAÇÕES TÉCNICAS

Para lhe dar algumas orientações iniciais, por favor, certifique-se de que implementa:

- Redux para armazenar dados (se necessário)
- Hooks em vez de Classes

Por favor, utilize a seguinte estrutura inicial de pastas:

/componentes para o seu material personalizado
/actions para dispatched actions  e pedidos de rede
/redux para os reducers e store
/páginas onde o conteúdo real da página estara
